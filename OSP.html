Guardian of Geist

Guardian of Geist is currently a tech demo version of a roguelite game. Play as a firey warrior, taking on a dungeon of ghouls and wizards. Current plans for the future development of the game will see randomly generated maps, passive stat-based and active ability items, giant boss fights, as well as more playable characters joining the roster.

Development Process

Guardian of Geist is a game currently being created as a solo project. Development has been slow as most of the work has had to be undertaken between semesters of study (as seen in lengthy periods of inactivity in it's GitHub history). Despite this, the development of the game has given me the opportunity to put programming patterns and algorithms learned throughout each semester, into practice in a new and creative way. The tech demo is already a fun and exciting experience, so I'm greatly looking forward to where I might be able to take further development of this project in the future.

Challenges in Development

This project has been used to consolidate programming skills learned throughout different subjects over the course of my degree:

A* Pathfinding: The enemy AI pathfinding algorithm was written from scratch, and is responsible for enemy movement through a tree of nodes. This allows for more unique and engaging combat and interactions with enemy AI.

Cool-downs and simple HUD UI: This is essential in giving the player visual feedback from the UI of their ability usage. It also allows for players to strategically time their next upcoming attacks against enemies.

Tile Mapping and Map Creation: Visuals and backgrounds had to be layered around the player so that their movement in a 2D space appears 3D. This creates an engaging and "realistic" world for the player to experience.

Dynamic Node Connections: To be able to create new maps and areas quickly and easily without needing to manually connect nodes, the nodes are connected for pathfinding automatically if they are placed within the correct range of each other.
