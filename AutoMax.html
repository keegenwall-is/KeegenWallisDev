AutoMax Car Store

AutoMax is an online software application designed to streamline all stages of the car sales process for both users and staff. To achieve this, all features implemented into the system were designed with user-friendly and intuitive interfaces that allow customers to discuss, ask questions and wishlist different car models, book test drives and purchase cars, all from home. Similarly, the staff interface has a focus on increasing productivity by simplifying monotonous tasks, including writing and filtering end of day sales reports.

Development Process

AutoMax was created over a series of 3 sprints, R0 - R2, that using MVC and CRUD design patterns saw the development and documentation of 10 total features. Using the agile method of development, both functional and non-functional requirements were turned into user stories which allowed us to priotirize work to create a minimum viable product. User stories were added to an Azure project plan board with cost estimates, data requirements, constraints and risks all being documented to easily communicate with stakeholders. Interaction design models, data and application architecture models as well as a backend database model, allowed all developers to have a clear picture of the final goal of the SQL Database which was especially useful when merging branches on GitHub and managing database conflicts. Challenges in Development

As a part of the Advanced Software Design subject of my degree, this project came with many challenges, including:

Balancing dual-user experiences: AutoMax is a system for both customers and staff to use and so needed to be designed so that different features were accessible to some users and not to others, such as the sales report feature and all CRUD elements being only available for staff to interact with.

Database and Architecture Models: All architecture, interaction and database models were created with line notation that informs the reader of the cardinality of different database objects.

Unit Tests: Writing unit tests ensured that different functions in the Contorller of the MVC were working as intended.

SQL Workbench: Database conflicts were the most difficult merge conflicts to handle when managing different Git branches, but through a the main SQL file, the database was able to be joined and remade using SQL DROP, CREATE and INSERT comands.
